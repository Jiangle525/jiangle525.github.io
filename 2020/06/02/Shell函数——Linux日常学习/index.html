    1. Shell函数的好处

    1. 简化程序代码，实现代码重用

    • 实现一次定义多次调用。如：is_root_user（）函数可 以由不同的shell脚本重复使用。

    2. 实现结构化编程

    • 使脚本内容更加简洁，增强程序的易读性

    3. 提高执行效率

    • 将常用的功能定义为多个函数并将其保存在一个文件中，类似其他语言的“模块”文件
    • 在 ~/bashrc 里或命令行上使用 source 命令调用这个文件
    • 此文件中定义的多个函数一次性地调入内存，从而加快运行速度

    2. 函数的定义和调用

    1. 函数定义

    function 函数名 { commands }

    函数名 () { commands }

    2. 函数调用

    只需输入函数名即可调用函数，函数必须在调用之前定义

    函数名
    函数名 参数1 参数2 …

    3. 函数的存储和显示

    1. 函数的存储

    • 函数和调用它的主程序保存在同一个文件中

      函数的定义必须出现在调用之前

    • 函数和调用它的主程序保存在不同的文件中

      保存函数的文件必须先使用 source 命令执行，之后才 能调用其中的函数

    2. 函数的显示

    • 显示当前Shell可见的所有函数名 

      $ declare -F

    • 显示当前Shell可见的所有（指定）的函数定义 

      $ declare -f 
$ declare -f  <functionName>

    4. 函数的调用举例

    • 函数和调用它的主程序保存在同一个文件中 
    #!/bin/bash 
## filename: function-1.sh 
### User define Function (UDF) ### 
sql_bak () { echo "Running mysqldump tool..."; } 
sync_bak () { echo "Running rsync tool..."; } 
git_bak () { echo "Running gistore tool..."; } 
tar_bak () { echo "Running tar tool..."; } 
### Main script starts here ### 
PS3="Please choose a backup tools : " 
select s in mysqldump rsync gistore tar quit ; do 
    case $REPLY in
        1) sql_bak  ;; 
        2) sync_bak ;; 
        3) git_bak  ;; 
        4) tar_bak  ;; 
        5) exit     ;; 
    esac 
done
    • 函数和调用它的主程序保存在不同的文件
    #!/bin/bash 
## filename: function-2.sh 
### User define Function (UDF) ### 

sql_bak () { echo "Running mysqldump tool..."; } 
sync_bak () { echo "Running rsync tool..."; } 
git_bak () { echo "Running gistore tool..."; } 
tar_bak () { echo "Running tar tool..."; } 
    #!/bin/bash 
source /root/bin/function-2.sh 

### Main script starts here ### 
PS3="Please choose a backup tools : " 
select s in mysqldump rsync gistore tar quit ; do
    case $REPLY in 
        1|[mM]ysqldump) sql_bak  ;; 
        2|[rR]sync)     sync_bak ;; 
        3|[gG]istore)   git_bak  ;; 
        4|[tT]ar)       tar_bak  ;; 
        5) exit     ;; 
    esac 
done

    5. 参数与变量

    1. 参数

    • 调用函数时，使用位置参数的形式为函数传递参数
    • 函数内的$1-${n}、$*和$@表示接收到的参数
      • $1-${n}表示第n个参数
      • $*表示将所有位置参量看成一个字符串
      • $@表示将每个位置参量看成单独的字符串（以空格间隔）
    • 函数调用结束后位置参数$1-${n}、 $* 和 $@ 将被重置为调用函数之前的值
    • 在主程序和函数中，$0始终代表脚本名

    2. 变量

    • 函数内使用 local 声明的变量是局部变量

      局部变量的作用域是当前函数以及其调用的所有函数

    • 函数内未使用 local 声明的变量是全局变量

      主程序和函数中的同名变量是一个变量（地址一致）

    6. 函数的结束和返回值

    • 当函数的最后一条命令执行结束，函数结束
      • 函数的返回值就是最后一条命令的退出码
      • 其返回值被保存在系统变量$?中
    • 可以使用return 或exit显示地结束函数
      • return [N]
        • return将结束函数的执行
          • 可以使用N指定返回值
      • exit [N]
        • exit将中断当前函数和当前Shell的执行
          • 可以使用N指定返回值

    return只能返回整数值，可以使用标准输出实现函数的返回值

    • 标准输出实现函数的返回值
      • 通用的方法，既能返回整数又能返回字符串
      • 函数结束前使用 echo 命令将结果显示到标准输出
      • 调用函数时使用如下的格式将函数的输出结果存到变量RES中，之后便可使用变量$RES的值

    7. 标准输出实现函数的返回值举例

    #!/bin/bash 
# User define Function (UDF) 
to_upper () { 
    local str="$@" 
    local output 
    output=$(tr '[a-z]' '[A-Z]'<<<"${str}") 
    echo $output 
} 

### Main script starts here ### 
to_upper "This Is a TEST" 
res=$(to_upper "$@") 
echo "$res" 

res=$(to_upper "$1") 
[[ $res == "YES" ]] && echo "Continue..." || echo "Stop"
